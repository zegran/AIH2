### Concern 5 — consistency_gate.py too weak

**CONFIRMED.**

Current weaknesses:
1. **Single concatenated string check**: `all_tex()` concatenates all .tex files. The pattern
   `approx.*?72` passes as soon as `72` appears ANYWHERE in ANY section — discussion alone satisfies it,
   even if abstract or results were wrong.
2. **No per-section check**: the gate does not assert that 72% appears in `main.tex` (abstract/highlights)
   or in `results.tex` specifically.
3. **No figure check**: `fig2_variance_data.csv` conditional returns `[]` if file missing — it always
   skips since no such file exists. No check that the figure script hardcodes or outputs the right values.
4. **Figure-catalog.md not checked**: stale catalog with 0.55/0.33 is never flagged.
5. **Forbidden pattern for 33% is regex-complex and fragile**: the lookbehind/lookahead for historical
   transition context is a maintenance burden.

### B4 — Strengthen consistency_gate.py

**Problem:** gate checks concatenated string; no per-section assertion; no figure-catalog check;
fragile lookbehind patterns for forbidden values.

**Fix — rewrite with per-section checks:**
```python
# Load each file separately
main_tex = read(PAPER / "main.tex")          # abstract + highlights
results_tex = read(PAPER / "sections" / "results.tex")
discussion_tex = read(PAPER / "sections" / "discussion.tex")

# Per-section assertions
assert_contains(main_tex, r"72", "abstract/highlights must contain 72%")
assert_contains(results_tex, r"0\.717", "results must contain R²=0.717")
assert_contains(results_tex, r"0\.425", "results must contain R²=0.425")
assert_contains(results_tex, r"0\.081", "results must contain R²=0.081")

# Figure-catalog check
catalog = read(FIGURES / "figure-catalog.md")
assert_contains(catalog, r"0\.717", "figure-catalog must contain 0.717")
assert_not_contains(catalog, r"(?<!→)0\.55(?!\d)", "figure-catalog must not contain stale 0.55")

# Forbidden values (simpler, no lookbehind)
assert_not_in_main_text(r"(?<!\d→)0\.553", "stale joint R²=0.553")
```

**Objective verification:**
- `uv run python tools/consistency_gate.py` → PASS with the above assertions
- Introduce a synthetic error (edit 0.717 to 0.71) → gate FAILS
- Restore → gate PASSES again
